핵심 요구사항 분석

  • 운영 안정화 (1순위) — 접수 시작/마감, 결과 발표, 추가 제출 등 피크타임 집중 모니터링 및 AWS 스케일링으로 트래픽 급증 시 서버 다운 방지
  • 소/중규모 기능 개발 및 텍스트 변경 — 공고명·접수/마감/결과 안내·FAQ·이메일/SMS 문구 업데이트, 채용 차수별 지원서 필드 추가/수정 및 검증 규칙 조정, 추가 제출 항목(면접복·희망근무지 등) UI/API 연동, 전형별 운영 화면 보완 및 엑셀 다운로드/업로드 컬럼 조정
  • 버그 수정 및 이슈 대응 — 지원서 임시저장/최종제출·결과 조회 오류, 파일 업로드/다운로드, 본인인증 콜백, 이메일/SMS 발송 이슈, 게시판(공지·Q&A·이의신청) 권한/표시 오류
  • 인프라 운영 지원 — AWS 서버 모니터링, DB 조회/수정 지원, SQS 큐 및 Serverless 배치 실행 상태 점검, 배포·롤백 지원 및 운영자 안내
  • 과업관리·커뮤니케이션 (전담 PM) — 채용 대행사/운영사와의 요구사항 수합, 릴리스 노트 작성, 변경 요청 채널 운영
  • 보안/인증 — 지원자 민감 개인정보 취급에 따른 보안 수칙 준수 및 데이터 접근 제어

제안 포인트

1. 동일 기술 스택 운영 경험으로 인수인계 리스크 최소화 — NestJS 백엔드 + React/Next.js 프론트엔드 + AWS 조합의 대규모 서비스를 다수 운영한 경험으로, 착수 후 2주 내 신속한 인수인계와 공고 텍스트 수정이 가능합니다.

2. 채용 캘린더에 맞춘 탄력적 운영 체계 — 마감일·발표일 등 업무 집중 구간을 사전에 식별하여 피크타임 집중 모니터링 체계를 구성하고, 평시에는 기능 개선·버그 수정에 역량을 배분합니다.

3. 전담 PM + 개발 인력 분리 투입 — 과업관리·커뮤니케이션을 전담하는 PM이 채용 대행사 채널을 운영하고, 개발 인력이 코드 변경에 집중하여 일정 준수와 산출물 완성도를 동시에 달성합니다.

4. 대규모 트래픽·대량 엑셀 처리 운영 노하우 — 수백 개 API 엔드포인트와 대규모 데이터를 다루는 서비스를 운영하며 축적한 모니터링·장애대응 노하우를 본 프로젝트에 적용합니다.

PRD (운영 범위 정의서)

본 사업은 신규 제품 개발이 아닌 운영 중 채용 시스템의 유지보수·운영 안정화 사업으로, 운영 범위를 명확히 정의합니다.

1. 프로젝트 배경 및 목적

상반기 채용에 활용된 채용 운영 시스템을 대상으로, 하반기 채용 진행을 위한 부분 개선·유지보수·운영 안정화·배포 및 장애 대응을 수행합니다. 대상 시스템은 NestJS 백엔드, React 기반 운영/관리 프론트엔드, Next.js 기반 지원자 채용 프론트엔드, AWS 연동, 본인인증, 이메일/SMS, 게시판(공지·Q&A·이의신청), 파일 업로드/다운로드, 운영 배치 스크립트로 구성됩니다.

3. 기능 요구사항

3.1 운영 및 모니터링 지원

  • [필수] 접수 시작/마감, 결과 발표, 추가 제출 기간 등 피크타임 집중 모니터링 및 안정화
  • [필수] 운영 데이터 확인, DB 조회/수정 지원, SQS 큐 및 Serverless 배치 실행 상태 점검
  • [필수] 시스템 배포, 롤백 지원 및 운영자 안내

3.2 소/중규모 기능 개발 및 텍스트 변경

  • [필수] 공고명·접수/마감/결과 안내·FAQ·이메일/SMS 발송 문구 등 텍스트 업데이트
  • [필수] 신입직/전문경력직 채용 차수별 지원서 필드 추가/수정 및 검증 규칙 조정
  • [필수] 새로운 추가 제출 항목(면접복, 희망근무지 등) UI/API 연동
  • [필수] 전형별 운영 화면 보완 및 엑셀 다운로드/업로드 컬럼 조정

3.3 버그 수정 및 이슈 대응

  • [필수] 지원서 임시저장/최종제출, 결과 조회 관련 오류 수정
  • [필수] 파일 업로드/다운로드, 본인인증 콜백, 이메일/SMS 발송 이슈 대응
  • [필수] 공지사항·Q&A·이의신청 게시판 권한 및 표시 오류 수정

대응성채용 시즌 특성상 특정 기간 업무 집중에 대한 탄력적 대응

5. 기술 제약사항

기존 시스템 스택(TypeScript · NestJS · React · Next.js · MongoDB · AWS S3/SQS/Serverless)을 유지하며, 본인인증·이메일/SMS·클라우드 등 기존 외부 사업자 연동을 그대로 활용합니다. 신규 사업자 전환은 수행하지 않습니다.

6. 범위 외 사항 (Out of Scope)

  • 기존 시스템의 전면 재개발
  • DB 마이그레이션
  • 신규 사업자(본인인증, 클라우드 등) 전환
  • 과업 범위를 초과하는 대규모 신규 기능 개발(별도 협의)
