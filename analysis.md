## 3. 실현 가능성 분석 (내부용)

- **프로젝트 유형**: 운영 중 서비스 유지보수 retainer (FE 중심 웹 + BE 보조) → **강력 추천** 영역. 신규 빌드가 아닌 기존 시스템 부분 개선.
- **기간 특성**: 180일은 개발 공수가 아니라 하반기 채용 캘린더(접수→마감→발표)에 고정된 운영 기간. 압축 대상이 아니므로 **180일 그대로 유지**.
- **인력**: 전담 PM(과업관리·커뮤니케이션) + 개발 인력. 다수 인력 투입으로 피크타임 모니터링·장애대응과 기능개선 병행. 월 평균 ~15 M/D, 6개월 ~90 M/D 투입.
- **판정**: feasibility 양호. 기간 유지, 가격 경쟁력 위주 책정.

## 4. 포트폴리오 매칭

매칭 카테고리: "SaaS B2B" / "기업용 포탈·어드민" → 추천 조합 EZ-Approve · Series-B · Harmony Link (모두 NestJS/Next.js/AWS 동일 스택).

| 프로젝트 | 매칭 점수 근거 |
|---------|----------------|
| **Series-B** | NestJS 10 + Next.js 13 + AWS, 14개월 장기 운영, 200~300 API, 본인인증(KYC)·민감정보·대규모 데이터 → 운영/장애대응/보안 직결 (최상위 매칭) |
| **EZ-Approve** | NestJS + Next.js + TypeScript, 운영/관리 어드민(React), RBAC 권한·표시 제어, 게시판, 파일 처리 → 관리자 측 기능개선·게시판 권한 오류 수정과 일치 |
| **Harmony Link** | NestJS + Next.js + React + AWS CDK, 140+ API, 멀티테넌트, 알림(이메일/SMS), 인프라 운영·배포 → 인프라 운영·장애대응과 부합 |

MongoDB는 보유 포트폴리오(MySQL/PostgreSQL 위주)와 직접 일치하지 않으나, 핵심 차별점인 NestJS/Next.js/AWS/대규모 트래픽 운영으로 충분히 커버.

## 5. 최종 제안 요약

- **지원 금액**: 34,000,000원 (VAT 별도) — 클라이언트 예상 40M의 85%, 플랫폼 평균 입찰가(34.4M)와 동일선
- **지원 기간**: 180일 (계약 후 즉시 착수, 하반기 채용 시즌 연동)
- **핵심 제안 포인트**:
  1. 동일 스택(NestJS·React·Next.js·AWS) 대규모 운영 경험으로 인수인계 리스크 최소화 (2주 내 인수인계)
  2. 채용 캘린더 연동 탄력 운영 — 마감·발표 피크타임 집중 모니터링 + AWS 스케일링
  3. 전담 PM + 개발 인력 분리 투입으로 일정 준수·산출물 완성도 동시 달성
  4. 대규모 트래픽·대량 엑셀 처리 운영 노하우 적용
